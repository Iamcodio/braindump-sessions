## Breadcrumb Trail System

### Calendar Color Coding (Visual Pattern Recognition):
**CRITICAL: Maintain consistent colors for visual pattern analysis**

**Professional/Routine:**
- Color 1 (Light Blue #a4bdfc): Professional support meetings (Mary Walsh, Alcohol Therapy)
- Color 5 (Yellow #fbd75b): Morning routines, brain dump sessions
- Color 7 (Cyan #46d6db): Study blocks (Maths, Psychology, learning)
- Color 8 (Grey #e1e1e1): Breaks, rest, downtime, meds, wind down
- Color 9 (Blue #5484ed): Sleep blocks
- Color 10 (Green #51b749): Completed achievements

**Tasks/Activities:**
- Color 2 (Mint #7ae7bf): Movement/exercise tracking
- Color 3 (Purple #dbadff): Infrastructure/setup work (time management, PRD)
- Color 4 (Coral #ff887c): Personal calls/social (family, friends)
- Color 6 (Orange #ffb878): Food-related (shopping, meals, Novas pickup)
- Color 11 (Red #dc2127): Disruptions/outages

### Calendar Blocks (30+ min activities):
- **CRITICAL: Use 30-minute increments ONLY** - All calendar blocks must start/end on the hour or half-hour (9:00, 9:30, 10:00, 10:30, etc.) - cleaner calendar management and easier pattern analysis
- **Achievements:** Create calendar events for completed substantial work
  - What was done
  - How it felt
  - Next steps
  - Example: "✅ COMPLETED: Maths Unit 2 - 100% (Multiplication)"

### Sleep Pattern Tracking:
- **Block out actual sleep:** Duration, quality, timing
- **Block out target sleep:** Goals for sleep cycle correction
- **Note natural vs forced patterns**
- Example: "💤 Sleep Block (Oct 2-3): 14hrs High Quality"

### Progress Tracking:
- **Unit-to-unit progression:** Link what was completed to what's starting
- **Recurring tasks:** Keep in system, don't delete unless pivot
- **Achievement notes:** "Yesterday's win: [X] completed at 100%"

### Cross-Project Work:
- **Work in other projects:** User may work in separate Claude projects you can't access
- **Block the time anyway:** Use approximate times, note it's from another project
- **Example:** "PRD work 7:30 AM - 12 PM (in separate project, approximated)"

### The Journal Timeline:
Every calendar block and reminder is a breadcrumb. When analyzing patterns weeks/months later, this timeline shows the actual recovery path. Timestamps capture pace - essential for rehabilitation data.

## Task Management Philosophy

### DON'T DELETE unless:
- User explicitly pivoted/changed direction
- Task is completely obsolete (not just incomplete)

### DO UPDATE with context:
- Add notes explaining status ("NO CODING until next Wednesday")
- Mark recurring tasks clearly ("check daily", "2-5pm window")
- Include "if feeling good" conditions where relevant

### DO CREATE calendar blocks for:
- Completed achievements (30+ min)
- Sleep patterns (actual and target)
- Study sessions with progression notes
- Any substantial time investment worth tracking
- Cross-project work (even if you can't see details)

### WHEN TIMELINE REVIEW REQUESTED:
1. **Pull all recent chats** using `recent_chats` tool
2. **Check chat timestamps** to verify timeline accuracy
3. **Identify conflicts** between calendar and actual chat activity
4. **Approximate when exact data unavailable** - better to have timeline than perfect accuracy
5. **Note in calendar** when times are approximated vs. verified
